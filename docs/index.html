Overview

Tech Docs • Clear • Reusable

DataUtility is a lightweight, modular Python utility library built on top of pandas to simplify data cleaning, exploration (EDA), and preprocessing. You use one class: DataTools. Internally, methods are organized into modules.

Why this library exists

Most projects repeat the same tasks: inspect datasets, handle missing values, remove duplicates, detect outliers, encode categories, scale numeric features, and create derived features. DataUtility keeps these steps consistent and reusable.

What “safe” means here

Methods validate columns and types, avoid silent behavior, and support inplace=True/False so you choose between modifying dt.df or returning a copy.

Quick Start
Install requirements and create your first DataTools instance
⌄
Put these files inside docs/ (example: docs/index.html). Open index.html in your browser.
1) Install 
pip install pandas numpy
2) Import + create DataTools 
import pandas as pd
from data_utility import DataTools

df = pd.read_csv("data.csv")
dt = DataTools(df)
If you use a package structure, export DataTools from data_utility/__init__.py.
Common workflow: overview() → fix missing/duplicates → outliers → encoding → scaling → features.
Project Structure
Modular files combined into one DataTools class
⌄
 
data_utility/
├── __init__.py
├── data_tools.py        # main public interface (DataTools)
│
├── base.py              # shared helpers and validation
├── io_tools.py          # data loading utilities (optional)
├── eda.py               # inspection and overview
├── missing.py           # missing values and duplicates
├── outliers.py          # IQR and Z-score logic
├── encoding.py          # encoding helpers
├── scaling.py           # scaling utilities
├── features.py          # feature engineering
├── text_cleaning.py     # text preprocessing
└── types.py             # dtype transformations
Implementation idea: your DataTools class imports and “mixes” feature modules. That keeps your code organized while usage stays simple.
How modules connect

Most functions operate on self.df. Validation helpers live in BaseTools so every module can reuse the same checks.

Design Principles
Consistency rules used across every method
⌄
Single entry point Use DataTools as the only class you instantiate.
All methods operate on self.df Internal modules assume self.df exists and is a pandas DataFrame.
inplace=True / False is respected If inplace=True, update dt.df. If false, return a new DataFrame.
Validation everywhere Clear errors for missing columns, wrong dtype, or unsupported strategy names.
Predictable output Return DataFrame / Series / (df, mapping) exactly as documented—no surprises.
How to Use DataTools (Recommended Workflow)
When to use each group of methods + a simple real workflow
⌄
Think of DataUtility like a preprocessing toolbox. You don’t use everything every time. You pick what your dataset needs.
Step 1 — Inspect first (always)

Use overview(), head(), tail(), insights(), and missingReport() to understand shape, columns, nulls, and basic stats. 

summary, info = dt.overview()
print(summary)
print(info)
dt.missingReport()
Step 2 — Fix missing values + duplicates

If missing values are small: fill them. If a column is mostly missing: consider dropping. Use duplicates removal when repeated rows exist (common in merged datasets). 

dt.fillMissingValues("median", columns=["age"])
dt.dropDuplicates(columns=["email"], keep="first")
Step 3 — Handle outliers (only if needed)

Use outliers when your numeric columns have extreme values. For ML, clipping is often safer than dropping rows. 

# Quick outlier check
mask = dt.outlier_mask_iqr()
print("Outlier rows:", mask.sum())

# Safer: clip instead of dropping
dt.clip_outliers_iqr(inplace=True)
Step 4 — Encode categories

If you have text categories (city, dept), convert them to numbers. Use one-hot encoding for most classical ML models. 

dt.oneHotEncode(columns=["city", "dept"], drop_first=True)
Step 5 — Scale numeric features (only when needed)

Scaling is useful for models that are sensitive to feature magnitude (like logistic regression, SVM, KNN, gradient descent). 

dt.scale("standard", columns=["age", "salary"])
If you will train ML, split train/test first. Then fit transformations on train and apply to test. (For now your utility is simple—later you can add pipeline helpers.)

Missing Values and Duplicates
fillMissingValues(), dropMissingValues(), fillMissingGroupBy(), dropDuplicates()
⌄
 
dt.fillMissingValues("mean")
dt.fillMissingValues("median", columns=["age"])
dt.fillMissingValues("mode", columns=["city"])
dt.fillMissingValues("value", value=0)
# Drop rows with too many missing values
dt.dropMissingValues(axis=0, thresh=5)

# Drop columns with too many missing values
dt.dropMissingValues(axis=1, thresh=100)
# Fill salary per department using median (default)
dt.fillMissingGroupBy("department", "salary")

# Or use mean or mode
dt.fillMissingGroupBy("department", "salary", strategy="mean")
dt.fillMissingGroupBy("department", "city", strategy="mode")
How to choose fill strategy:
  • Mean: good if data is symmetric (no extreme outliers)
  • Median: safer if there are outliers (salary, prices)
  • Mode: best for categorical columns (city, dept)
  • Group-based: best when groups behave differently (salary depends on department)
Duplicate removal supports full-row duplicates or subset-based duplicates. 
dt.dropDuplicates()
dt.dropDuplicates(columns=["email"])
dt.dropDuplicates(columns=["email"], keep="last")
dt.dropDuplicates(columns=["email"], keep=False)
Outliers
IQR and Z-score: masks, filtering, and clipping
⌄
 
# Detect outlier rows (IQR)
outliers = dt.detect_outliers_iqr()

# Get mask for outliers (IQR)
mask = dt.outlier_mask_iqr()

# Clip outliers (IQR)
dt.clip_outliers_iqr(inplace=True)
# Detect outlier rows (Z-score)
outliers = dt.detect_outliers_zscore(z=3)

# Get mask for outliers (Z-score)
mask = dt.outlier_mask_zscore(z=3)

# Clip outliers (Z-score)
dt.clip_outliers_zscore(z=3, inplace=True)
Use outliers when:
  • Numeric columns have extreme values (salary, revenue, weights)
  • Your model becomes unstable or metrics are weird
  • You see very large std / skew
Prefer clipping for ML to reduce data loss.
Encoding
oneHotEncode() and labelEncode()
⌄
 
# One-hot encode (auto-detect categorical columns)
dt.oneHotEncode()

# One-hot encode specific columns
dt.oneHotEncode(columns=["city", "dept"], drop_first=True)

# Label encode one column (optional mapping)
df_encoded, mapping = dt.labelEncode("category", return_mapping=True)
Tip: Label encoding turns categories into integers. For most ML models, one-hot is safer unless you are sure the model handles ordinal meaning correctly.
Scaling
scale() with standardization and min-max
⌄
 
# Standardize all numeric columns
dt.scale("standard")

# Min-max scale selected columns
dt.scale("minmax", columns=["age", "salary"])
Scaling affects numeric columns only. Constant columns (std=0) should be skipped safely.
Feature Engineering
combineFeatures() using expressions on existing columns
⌄
 
dt.combineFeatures("profit", "revenue - cost")
dt.combineFeatures("bmi", "weight / (height ** 2)")
dt.combineFeatures("full_name", "first + ' ' + last")
Expressions run using pandas DataFrame.eval(..., engine="python"). If a column name is wrong, you should raise a clear error.
Text Cleaning
cleanText() for basic normalization
⌄
 
dt.cleanText(
    ["review_text"],
    lower=True,
    strip=True,
    remove_punct=True,
    remove_extra_spaces=True
)
This stays intentionally simple. Later you can add stopwords, stemming, tokenization, or sklearn pipelines.
Type Conversion
to_int(), to_float(), to_string(), to_bool(), to_datetime(), to_category()
⌄
 
dt.to_int("age")
dt.to_float("price")
dt.to_datetime("date")     # invalid values become NaT
dt.to_category("city")
Conversions can fail if values are incompatible (example: non-numeric strings → int). Clean values first if needed.
